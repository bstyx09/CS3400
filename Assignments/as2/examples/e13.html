Cross Browser

Cross Browser and Device Techniques



There is a wide variety of devices that can be used to access internet content, and many of those devices have smaller screens. Traditional layouts for larger computer screens often do not translate well to a smaller screen, leading to headaches when trying to present content in a uniform manner to more than one platform. Fortunately, there are a few different ways CSS can help.



Media Queries


Media queries are queries from the server to the local browser to determine information that allow the server to determine the best method to present the internet content to the platform or browser in question.

One example of this is to test the device width.



<link rel="stylesheet" media="screen and (min-device-width: 800px)" href="800.css" />

The above uses the '800.css' stylesheet only if the viewing device has a width of 800px or wider.

Another method of doing this is to test for the browser width instead of the device width. After all, browser windows are not always maximized to the full screen dimensions. This is called testing for the 'viewport.'


<link rel='stylesheet' media='screen and (min-width: 701px) and (max-width: 900px)' href='css/medium.css' />

The above stylesheet only takes affect when the browser window is between 701 and 900 pixels in width.

The browsers that support media queries include IE 9+, Firefox 3.6+, Chrome and Opera 10+.
