Browser Compatibility



While all major browsers support JavaScript, they do so to varying degrees, and in different ways. As a result, a piece of JavaScript code that may work fine in one browser might do nothing in another. As a result, developers should do several things when writing code:


  1. Check which browsers support the functions and libraries you are using. A fantastic resource for this is this compatibility table, referenced from Mozilla's excellent web developer resource.

  2. Check to ensure that your code only executes on browsers with full function support. There are a number of ways this can be done. The first involves checking for which browser is being used, then handle each instance separately.

         if (navigator.userAgent.indexOf("MSIE 8") > -1) {
            // do IE8 stuff
         }
         else {
            // do non IE8 stuff
         }

    Unfortunately, the user agent strings tend to not be stable, and besides, the method itself is too static to really be considered good design, as the code would have to be modified, or at least tested, against every likely browser version. Anyone running something unanticipated might encounter some nasty results.

    Fortunately, there is another method that involves testing whether or not the browser supports the feature in question. As a result, instead of running tests for a number of different browsers, one simple if else test to check for function support follows:

         if (typeof console === "undefined" || typeof console.log === "undefined") {
            // do nothing, or use an alternative like 'alert()'
         }
         else {
            // use console.log without worry
         }

  3. Finally, there are tools available, both free and paid, that will check your own code for compatibility with different browsers, much like an HTML validator. One of the free ones available to use include Browser Shots.
