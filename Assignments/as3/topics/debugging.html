Debugging



JavaScript is extremely difficult to write beyond the most trivial applications without a debugger. It can be difficult to glean errors just by sifting through the code, and typically code that contains errors will simply not run, and throw no error messages.

Fortunately, all modern browsers have a built-in debugger that can be toggled on and off. (Note: Many IDEs, including the ones discussed in the IDE section, come bundled with a debugger.) Debuggers allow the user to set breakpoints within the code in order to examine variables during execution, which is often of great assistance.

In order to make use of a debugger in the code, the console.log() method will display values in the debugger window:


     <!DOCTYPE html>
     <html>
     <body>

     <h1>My First Web Page</h1>

     <script>
     a = 5;
     b = 6;
     c = a + b;
     console.log(c);
     </script>

     </body>
     </html>

In addition to setting breakpoints via the debugger, you can code breakpoints into JavaScript by using the debugger keyword, which will stop the execution of JavaScript at the keyword location, and call the debugging function if it's available.
