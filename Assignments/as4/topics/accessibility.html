Accessibility



Accessibility focuses on how a disabled person accesses or benefits from a site, system or application. This is an important part of designing your site and should be considered throughout the development process. (Usability.org)

But why is this important? According to W3.org, accessibility for users of all levels is a basic human right which supports social inclusion, and also makes good business sense. Additionally, accessibility tends to go hand in hand with best practices for design philosophies for mobile web sites, device independence, multi-modal interaction, usability, design for older users and search engine optimization.


Best Practices


So now that we have shown the value for designing with accessibility in mind, how do we implement such content? A fantastic guide for doing so is the Web Content Accessibility Guidelines 2.0, last updated September 16, 2014. Some basic examples of accessible feature implementation follow.


Alternative Image Text


Alt text for images is a simple way to duplicate the basic information from an image in an accessible way for users who can't view the image. (This is not limited to disabilities, but also includes constraints of infrastructure such as rural or third world locations that don't have much bandwidth, as just one example.)

Alt text allows screen readers to relay the image information to people who can't see the screen, as well as make it available to technologies such as search engines.




This is simple to do, yet keeps a user's experience relatively consistent regardless of his or her ability to view images.


Color Awareness


In a similar vein, while color is a wonderful stylistic element to a user interface, a good design should never rely solely on color differentiation as a navigation tool or indicator, as color blindness makes such features much less useful. Instead, use more explicit differentiation, or perhaps reinforce color design with information of another form.


Keyboard Input


Not everyone can use a mouse, and providing functionality for keyboard-only input or some combination therein helps to include a larger user base. This also assists users who may not be able to fully use a keyboard, but can use speech input with assistive technologies that mimic a keyboard.


Audio/Video Transcripts


Audio and video content are everywhere, and while not everyone can experience them in that form, text transcripts broaden those who can interact with those elements of a site. Transcripts also provide search engines with information about the content.


Secondary Experience


The design of accessibility in a website often is thought of as secondary to the main design. While in some ways this is true, depending on the user base, that doesn't mean the disabled user should have a lesser experience to the standard user. Though the design may be simplified or distilled down to basics from some untenable features, the overall experience of users with disabilities should be functional, rich and enjoyable.
