Canvas



The <canvas> is an HTML5 element that can be used to draw graphics with scripting. Typically, the scripting involves is JavaScript, and indeed, the largest libraries are JavaScript based. It is well supported in recent versions of all major browsers.

The Mozilla developer resource provides a wonderful tutorial for the use of the <canvas> element.


KineticJS



There are a number of popular libraries or frameworks for <canvas>, but the most widely used one seems to be KineticJS. It has a rich feature list, event support and an active community. The principle of the framework follows directly from their wiki:

Kinetic stages are made up of user defined layers. Each layer has two canvas renderers, a scene renderer and a hit graph renderer. The scene renderer is what you can see, and the hit graph renderer is a special hidden canvas that's used for high performance event detection. Each layer can contain shapes, groups of shapes, or groups of other groups. The stage, layers, groups, and shapes are virtual nodes, similar to DOM nodes in an HTML page. ...

All nodes can be styled and transformed. Although KineticJS has prebuilt shapes available, such as rectangles, circles, images, sprites, text, lines, polygons, regular polygons, paths, stars, etc., you can also create custom shapes by instantiating the Shape class and creating a draw function.

Once you have a stage set up with layers and shapes, you can bind event listeners, transform nodes, run animations, apply filters, and much more.

There is even a fun demo available where you can see the source code.
