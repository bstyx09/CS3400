HTML Standards



Closing Elements


All tags should be closed to denote the closing of an element.
Self closing tags should have a space before the forward slash.


<br />


Attributes and Tags


Every attribute and tag should be written in lowercase. However, if the data in the code needs to be read by a human, proper capitalization should be followed.


<a href="http://www.google.com" title="Google website link">Google.com</a>


Quotes


In order to follow W3C specifications for XHTML, every attribute should have a value and use double or single quotes
Failure to use quotes could lead to security vulnerabilities.


Incorrect:
				
<input type=text name=email disabled=disabled>
				
Correct:
				
<input type="text" name="email" disabled="disabled" >
<input type='text' name='email' disabled='disabled' >


Indentation


When indenting, tabs should be used instead of spaces.
Nested elements should be indented accordingly. PHP blocks should also be indented to match the HTML code surrounding it.
Closing blocks should match the indentation of the opening blocks.

Correct:

<?php if ( ! have_posts() ) : ?>
	<div id="post-1" class="post">
		<h1 class="entry-title">Not Found</h1>
		<div class="entry-content">
			<p>Apologies, but no results were found.</p>
			<?php get_search_form(); ?>
		</div>
	</div>
<?php endif; ?>


Incorrect:

		<?php if ( ! have_posts() ) : ?>
		<div id="post-0" class="post error404 not-found">
			<h1 class="entry-title">Not Found</h1>
			<div class="entry-content">
			<p>Apologies, but no results were found.</p>
<?php get_search_form(); ?>
			</div>
		</div>
<?php endif; ?>
