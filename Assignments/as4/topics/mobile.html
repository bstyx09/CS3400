Mobile Interface Design


Web-enabled mobile devices are becoming increasingly prevalent, and understanding their strengths and limitations is the key to creating well designed interfaces with these devices in mind. W3.org specifies standards for these devices, which have unique constraints.

As W3.org points out, a number of the barriers that mobile users face are similar to those of users with disabilities. For example, the lack of mouse or keyboard functionality is often an overlapping concern.

Some helpful resources for mobile design include mobilexweb.com, which has a list of official user interface and user experience guidelines from the manufacturers of mobile devices. Key among these are the iOS design principles and the Android design principles. W3.org also supplies mobile web best practices and mobile web application best practices. Several of these follow:

  • Make telephone numbers click to call. This can be as simple as using a standardized telephone URI scheme. In this case:

         <a href="tel:[PHONE-NUMBER]">[PHONE-NUMBER]</a>

    e.g., +1-555-555-5555

    Note that the full international prefix is used so that it will work outside of its home country. After all, these devices are mobile.

  • Ensure paragraph text flows. Small screens make it important for text to be responsive to screen size so that no horizontal scrolling is required, and also so that it will re-flow if the view orientation is changed. In essence, this requires using percentage and relative measures:

    em, ex, bolder, larger, thick

    Avoid absolutes or pixels.

  • Keep DOM size reasonable. The in-memory size of the Document Object Model (DOM) may be limited on mobile devices. Exceeding this limit can result in unpredictable errors. Limiting the amount of information in the DOM at a single time using pagination (e.g., limiting google results to 10 per page) can prevent this from happening.

  • Use the meta viewport element to identify desired screen size. Some browsers attempt to display desktop pages on small screens by automatically zooming the display, which can interfere with other optimizations for smaller screens. The viewport meta tag tells the devices at what scale to render the page.

         <meta name="viewport" content="width=device-width, initial-scale=1.0"/>

    This is inserted into the <head> element of the HTML document, and prevents browser based scaling.

  • Offer users a choice of interfaces. Device characteristic detection is imperfect, and cannot always account for the different use cases of an application. As such, it's important to offer the user the choice to change the display version if more than one version exists. It's important when doing so to default to the most appropriate UI on first use, and to store/remember the selection preference for future visits, either in local data storage or in a cookie.

  • Use gestures. Many mobile devices do not have a dedicated pointing device, and typically a software keyboard that interferes with the display area. As such, bypassing the need for those with finger gestures to allow for swiping and dragging will improve the response time of your design for mobile users.
